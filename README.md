# 🐝 NagaBuzz Web Portal
> Independent Privacy Policy & User Account Deletion Compliance Portal for **NagaBuzz** (Nagaland's premier news, notice board, and discussion application).

This standalone frontend project is a premium single-page application built to satisfy **Google Play Console User Data Deletion Policy** and **Apple App Store compliance requirements**, providing a high-fidelity portal for your users.

---

## 🎨 Design System & Aesthetics
- **Drifting HSL Glows**: Premium custom keyframe animations that drift lazily across the background.
- **Glassmorphic Cards**: Frosted-glass containers (`backdrop-filter: blur(16px)`) with thin high-frequency borders.
- **Adaptive Dark/Light Theme**: A persistent dark/light theme switch cached directly in client storage (`localStorage`).
- **Interactive Search**: Non-destructive keyword filtering and text-node highlighting inside the Privacy Policy.
- **Digital Receipt Wizard**: A multi-step account deletion process culminating in a print-ready audit receipt and mock registry.

## 🔐 Supabase Environment & Table Setup

### Reusing the App Environment
This portal connects **directly** to the same live database instance as your mobile app. By sharing the credentials:
1. Deletion requests are recorded directly in the shared Supabase database.
2. Your backend triggers, automated scripts, or admin console can process these deletion tickets to wipe active user profiles from Firebase Auth and related database tables simultaneously.
3. The Supabase `anon` public key is completely safe to put in the client-side code because Supabase is designed to protect queries using **Row-Level Security (RLS)**.

### Database Setup SQL Script
To establish database synchronization, copy the PostgreSQL script below, go to your **Supabase Dashboard**, open the **SQL Editor**, paste it, and click **Run**:

```sql
-- 1. Create the Deletion Requests tracking table
CREATE TABLE IF NOT EXISTS public.deletion_requests (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    ticket_id VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(255) NOT NULL,
    auth_provider VARCHAR(100) NOT NULL,
    scopes TEXT NOT NULL,
    submission_date TIMESTAMPTZ DEFAULT NOW(),
    grace_date TIMESTAMPTZ NOT NULL,
    status VARCHAR(50) DEFAULT 'PENDING_AUDIT',
    reason VARCHAR(100) DEFAULT 'none',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. Enable Row-Level Security (RLS) to secure the table
ALTER TABLE public.deletion_requests ENABLE ROW LEVEL SECURITY;

-- 3. Create Insert-Only Policy for Public Visitors
-- Allows the web portal to insert requests directly, but blocks viewing or tampering with existing tickets
CREATE POLICY "Enable insert access for public portal submissions" 
ON public.deletion_requests 
FOR INSERT 
TO public 
WITH CHECK (true);

-- 4. Create Select Policy for Authenticated Service Role (Admins Only)
-- Allows your administrators and background triggers to query/process the requests
CREATE POLICY "Enable read access for authenticated admins only" 
ON public.deletion_requests 
FOR SELECT 
TO service_role 
USING (true);
```

## 🚀 Running Locally

This project is extremely lightweight and has **zero build dependencies**. 

### 1. Prerequisite
Ensure you have [Node.js](https://nodejs.org/) installed on your machine.

### 2. Startup Server
Navigate to this folder in your terminal and run the local development server:
```bash
# Install static http server globally/locally if not cached
npm install

# Run the dev server on http://localhost:8080
npm run dev
```

Open [http://localhost:8080](http://localhost:8080) in your browser to view and test the interactive portal.

## 🌐 Deploying to Production

Since this is a fully static project (HTML, CSS, JS), you can deploy it online **for free in under 5 minutes**:

### Option A: Vercel (Recommended)
1. Initialize a Git repository in this folder and push it to GitHub:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```
2. Go to [Vercel.com](https://vercel.com/) and click **Add New Project**.
3. Import your GitHub repository.
4. Leave all settings as default and click **Deploy**. Vercel will instantly host your compliance portal on a fast, global CDN!

### Option B: GitHub Pages
1. Push your code to a public GitHub repository.
2. In your repository settings, navigate to **Pages**.
3. Under **Build and deployment**, select **Deploy from a branch** and choose the `main` branch.
4. Save, and your site will be active at `https://<your-username>.github.io/<repo-name>`.

### Option C: Netlify
1. Go to [Netlify.com](https://www.netlify.com/) and drag-and-drop this folder directly into the Netlify Web App console.
2. Your site is instantly online!
